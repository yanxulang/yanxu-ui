# 言界（yanxu-ui）

言界是以言序代码为主体实现的保留模式桌面 GUI 框架。它只通过
[言台](https://github.com/yanxulang/yanxu-platform) 获取窗口、输入、文字、图片、系统服务和
整帧绘制原语；按钮、输入框、列表、标签页等高级控件全部由言序实现。

当前版本为 `0.9.0`，源码加载最低要求言序 `1.1.9`、言台 `1.0.x` 和言据 `1.1.x`。
正式 YXB 与 Bundle 使用经过回放验收的言序 `1.1.20` 构建；言包、言据和既有
`yanxu-gui` 无需升级或迁移。

## 安装

在应用的 `言序.toml` 中声明：

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.9.0", 版 = "^0.9" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

然后由言包安装并生成目标平台锁文件。开发本仓库时可以把依赖改为 `路径 = "../yanxu-ui"`。

## 快速开始

```yanxu
引「包:言界」为 界面；

定 应用 为 界面.应用（「你好」）；
定 窗口 为 应用.窗口（{「标题」：「你好，言界」，「宽」：640，「高」：420}）；
定 布局 为 窗口.列（{「内边距」：24，「间距」：12}）；
定 输入 为 布局.输入框（{「占位」：「请输入中文」}）；
定 提示 为 布局.文字（「尚未点击」）；
定 按钮 为 布局.按钮（「问候」）；

法 点击（所事件）则 提示.内容（「你好，」加 输入.取内容（））；终
法 关闭（所事件）则 应用.退出（）；终
按钮.点击（点击）；窗口.关闭时（关闭）；
窗口.显示（）；应用.运行（）；
```

## 已实现能力

- 应用、原生窗口、多窗口、计时器与系统主题协调；
- 控件树和组件关闭生命周期；
- 行、列、堆叠、网格、滚动及分割布局；
- 外边距、内边距、间距、伸展、最小／最大／首选尺寸；
- 命中测试、指针捕获、事件捕获／目标／冒泡；
- 焦点顺序、Tab／Shift+Tab 与应用快捷键；
- 浅色、深色、自定义主题、运行时缓存失效及状态样式继承；
- 保留渲染树、脏区合并和版本化二进制整帧提交；
- Unicode 单行／多行编辑、选区、剪贴、撤销／重做、上下/Home/End 与中文 IME；
- 文字、按钮、输入框、复选框、单选框、切换、下拉选择、滑块、进度条、滚动容器、虚拟
  列表、标签页、分割面板、菜单、弹出层、画布和图片；
- 窗口级叠层栈、弹层锚定与自动翻转、嵌套模态焦点范围、外部点击／Escape 关闭，以及
  子菜单和完整菜单键盘导航；
- Unicode 多行精确选区、按行高亮和有界滚动；有界增量数据源、虚拟列表、缓存淘汰、加载／失败
  状态和逐行无障碍选择；
- 可观察状态、数据绑定、单调时间动画值及带焦点/动作回传的原生无障碍语义树；
- 帧呈现反馈驱动的有界动画、单槽背压、版本化运行诊断和固定运行器性能预算；
- 言台资源配额协商、托管计时器与图片、单向窗口／应用关闭、结构化关闭报告和资源归零压力门禁；
- 六目标原生运行、全控件无障碍审计、长交互序列、资源归零和候选重放生产门禁；
- 1.0 候选 API 冻结、0.1.0–0.8.0 机器兼容证明、供应链证据和独立发布候选演练；
- 言据优先的主题配置和等价 JSON 兼容入口。

## 两条 GUI 路线

```text
现有路线：言序应用 → yanxu-gui（言窗）→ egui/eframe/winit → 操作系统
新路线：言序应用 → yanxu-ui（言界）→ yanxu-platform（言台）→ 操作系统
```

`yanxu-gui` 继续保持兼容。新应用可根据控件成熟度和可定制性选择路线；无需强制迁移。
