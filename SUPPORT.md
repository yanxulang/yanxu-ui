# 支持与稳定性

言界 1.0 是保留模式桌面 GUI 的首个稳定支持线。本文定义 1.x 的公开兼容范围、依赖边界、
修复策略和发布证据；API 的精确机器快照见`api/api-v1.freeze.json`。

## 支持线

| 版本线 | 状态 | 处理方式 |
| --- | --- | --- |
| `1.0.x` | 稳定支持 | 接收兼容修复、安全修复、文档与生产门禁修复 |
| 后续 `1.x` | 向后兼容演进 | 只增加可选能力；不删除或改签 1.0 公开面 |
| `0.x` | 迁移支持 | 保留迁移和兼容证明，不再增加功能；安全问题按影响单独评估 |

同一小版本线只发布向后兼容补丁。新的可选控件、方法或状态需要提升次版本，并同时更新 API
快照、兼容证明、示例和迁移说明。删除公开声明、收窄参数、扩大调用方必须处理的返回类型、
改变既有字段单位或复用事件名称，必须进入新的主版本。

## 1.0 公开承诺

- 保留冻结快照中的 30 个声明、23 个类、7 个包级函数、125 个域和 375 个方法；继承成员
  可以上移到父类，但旧调用方式必须继续成立；
- 保留 0.1.0 起已经发布的事件名称、主题字段、布局单位、控件创建方法和 JSON 兼容入口；
- 保留言台 ABI v2、平台 1.7、事件 1.3、无障碍 1.0 与绘制 1.1 的消费边界；同主版本新增
  的可忽略字段可以兼容，主版本不匹配时明确拒绝运行；
- 保留窗口、计时器和图片的单向生命周期、幂等关闭、结构化关闭报告以及资源配额语义；
- 保留六个正式桌面目标的源码兼容与原生运行门禁。

这些承诺适用于源码重新解析和重新构建。YXB、Bundle、目标锁、诊断快照和无障碍树修订不是
跨版本永久序列化格式；升级后应为每个目标重新生成锁并重建应用。

## 工具链与依赖

- 言序 1.1.9 是源码、API 生成和公开目标锁的兼容下限；
- 言序 1.1.20 是 1.0 Release YXB 的构建与实际回放基线；
- 言台使用稳定 `^1.0` 契约，1.0 发布固定到标签`v1.0.0`及其精确提交；
- 言据使用 `^1.1`，1.0 发布固定到已经审核的 1.1.2 提交；
- 应用必须提交各目标独立生成的锁，不得复制其他系统或架构的锁。

最低与发布工具链不是“只能使用”的上限。更新的兼容言序可以加载源码，但正式升级前仍应
运行应用自己的目标系统测试；言界 CI 同时保留最低工具链和当前稳定工具链门禁。

## 发布与修复

每个 1.x Release 必须来自已合并提交，依次通过主线 CI、同提交版本标签 CI 和 Release
来源核对。六目标归档、SHA-256、完整清单和 API 快照只复用标签 CI 已验收候选；发布阶段
不重新生成二进制。候选报告记录来源提交、API／兼容／供应链摘要、六套锁摘要和六个回放
YXB 的摘要与字节数。

兼容或安全修复应带最小回归测试。安全问题请使用 GitHub 私密安全公告；公开 Issue 不应
包含利用代码、用户内容、剪贴板数据、输入内容或原生句柄。没有公开响应时间保证，但确认的
高影响问题优先在当前稳定补丁线处理。

## 支持边界

1.0 支持 Windows、macOS、Linux GNU 的 x86-64 与 ARM64，并依赖目标系统的字体、输入法、
屏幕阅读器和桌面服务。GPU、移动端、Web、32 位目标、异步数据源调度、网络权限和全局快捷
键注册不属于 1.0 承诺；应用仍须对这些能力自行实现或选择其他路线。
