# 兼容性

## 版本

| 组件 | 兼容范围 | 0.8 验证版本 |
| --- | --- | --- |
| 言序 | 源码 `>=1.1.9`；正式制品 1.1.20 | 1.1.9、1.1.20 |
| 言包 | 能处理格式 2 清单与锁文件 | 0.5.0 |
| 言台 | `^1.0` | 1.0.0 |
| 言据 | `^1.1` | 1.1.2 |
| 言窗 | 独立并行路线 | 0.1.0，未修改 |

言界 0.8.0 的源码兼容下限为言序 1.1.9，以获得按宿主事件隔离的执行步数预算；生产
YXB 与 Bundle 基线为当前稳定版本 1.1.20。发布门禁在六个目标保留 1.1.9 源码、测试和
锁文件验证，并使用 1.1.20 隔离构建、重新打开和实际运行预编译候选。不要求言包 0.5.1
或新的言据版本。

言台 1.0.0 保留言界已经使用的 ABI v2、平台 1.7、事件 1.3 和绘制 1.1 兼容边界；依赖
升级只切换到稳定支持线和六目标正式制品，不改变言界公开控件 API 或主题配置格式。

## 平台

言界字节码本身不含平台条件分支，随言台支持下列目标：

| 目标 | 窗口／事件／CPU 绘制 | CI |
| --- | --- | --- |
| `x86_64-pc-windows-msvc` | 支持 | 正式矩阵 |
| `aarch64-pc-windows-msvc` | 支持 | 正式矩阵 |
| `x86_64-apple-darwin` | 支持 | 正式矩阵 |
| `aarch64-apple-darwin` | 支持 | 正式矩阵和真实窗口 |
| `x86_64-unknown-linux-gnu` | Wayland／X11 | 正式矩阵和 Xvfb 窗口 |
| `aarch64-unknown-linux-gnu` | Wayland／X11 | 正式矩阵 |

同一上层源码不按操作系统分支。实际字体像素、系统对话框外观和 IME 候选窗由系统资源决定。

## 兼容政策

- 1.0 前的小版本用 Release Notes 说明新增能力和升级边界；
- 0.8 保留 0.1–0.7 已发布的事件名称、主题字段、布局单位和公开控件方法，并增加有界滚动、
  列表与标签逐项语义、分割面板统一调整及生产后端验收；
- 言台事件／绘制主版本不匹配时拒绝运行，次版本新增字段会被忽略；
- 言据为首选可读格式，JSON 兼容入口不会在 1.0 前移除；
- 不会以新路线替换或废弃 `yanxu-gui`。

## 已知限制

- 数据源加载器是同步回调；异步调度、网络权限和取消由应用层负责；
- 无障碍树、焦点以及全部公开控件的合法角色、标量值、状态和动作组合已接入原生桥；
  系统辅助技术的具体表现仍受 UIA、NSAccessibility 或 AT-SPI 实现与系统配置影响；
- 菜单快捷键字段只负责显示，不自动注册全局快捷键；全局快捷键仍由窗口快捷键表负责；
- 列表只为当前可见范围生成最多 256 个语义节点；离开当前树的行身份不能用于后续动作；
- GPU 后端不是 0.8 的承诺，当前统一使用言台 CPU 二维后端。
