# 兼容性

## 版本

| 组件 | 兼容范围 | 0.1 验证版本 |
| --- | --- | --- |
| 言序 | `>=1.1.9` 且语言规范 1 | 1.1.9 |
| 言包 | 能处理格式 2 清单与锁文件 | 0.5.0 |
| 言台 | `^0.1` | 0.1.0 |
| 言据 | `^1.1` | 1.1.2 |
| 言窗 | 独立并行路线 | 0.1.0，未修改 |

言界 0.1.1 要求言序 1.1.9，以获得按宿主事件隔离的执行步数预算；不要求
言包 0.5.1 或新的言据版本。

## 平台

言界字节码本身不含平台条件分支，随言台支持下列目标：

| 目标 | 窗口／事件／CPU 绘制 | CI |
| --- | --- | --- |
| `x86_64-pc-windows-msvc` | 支持 | 正式矩阵 |
| `aarch64-pc-windows-msvc` | 支持 | 正式矩阵 |
| `x86_64-apple-darwin` | 支持 | 正式矩阵 |
| `aarch64-apple-darwin` | 支持 | 正式矩阵和真实窗口 |
| `x86_64-unknown-linux-gnu` | Wayland／X11 | 正式矩阵和 Xvfb 窗口 |
| `aarch64-unknown-linux-gnu` | Wayland／X11 | 正式矩阵 |

同一上层源码不按操作系统分支。实际字体像素、系统对话框外观和 IME 候选窗由系统资源决定。

## 兼容政策

- 0.1 次版本只增加公开方法、控件或可选配置字段；
- 现有事件名称、主题字段和布局单位在 0.1 系列内不改变含义；
- 言台事件／绘制主版本不匹配时拒绝运行，次版本新增字段会被忽略；
- 言据为首选可读格式，JSON 兼容入口不会在 0.1 系列移除；
- 不会以新路线替换或废弃 `yanxu-gui`。

## 已知限制

- 跨行选区背景还是保守矩形，文本模型与 IME 提交本身不受影响；
- 无障碍语义树已经生成，0.1 尚未接入各系统的原生无障碍桥；
- 菜单与弹出层提供基础行为，尚无子菜单、模态焦点陷阱和屏幕边缘自动翻转；
- 列表按可见行绘制，但 0.1 未公开异步数据源；
- GPU 后端不是 0.1 的承诺，当前统一使用言台 CPU 二维后端。
