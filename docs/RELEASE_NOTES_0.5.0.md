# 言界 0.5.0

言界 0.5.0 在已发布的言台 1.0 稳定契约上完成文本与数据视图能力。多行输入框现在按
Unicode 字符边界逐行绘制选区；虚拟列表通过有界数据源按可见范围加载、缓存和恢复，不把
异步数据或大集合展开成无界控件树。

## 新功能

- 文本模型增加字节范围到字符范围的严格往返校验、行数／行范围／行文本查询、按行选区段和
  字节选区快照；切分多字节 Unicode 字符的范围会被拒绝。
- 多行输入框按视口行逐行生成文字命令和选区高亮，跨行选择、中文、Emoji 和滚动使用同一
  套字符索引。
- 新增有界增量数据源：总数、批次、缓存、版本和加载状态均有明确上限；加载器返回数量
  不一致、非列或抛出错误时原子失败并保留稳定诊断。
- 新增虚拟列表：只请求可见范围，支持增量提供、范围失效、缓存清空、加载／失败占位、滚轮
  选择和辅助技术整数行选择；数据更新自动触发重新布局和脏区刷新。
- 数据源监听改用单调递增编号；虚拟列表切换数据源时按编号解除旧订阅，避免对象方法身份
  不稳定造成重复回调或旧数据源泄漏。
- 新增`虚拟数据视图.yx`公开示例与文本／数据视图指南。

## 修复内容

- 长文档不再在单次绘制中把全部换行文本和选区展开；选区几何和绘制命令数量受视口边界约束。
- 数据源失败不会留下部分写入的缓存；刷新会清除旧范围并允许应用重新尝试。
- 视图接收数据版本、数量变化和缓存清空事件时保持当前选择和焦点身份。
- 列表与虚拟数据视图实现从包入口拆分，最低工具链的正式构建不再超过单模块 32 MiB 上限。

## 兼容性与平台

- 最低源码工具链为言序 1.1.9；正式制品由言序 1.1.20 在隔离源码副本中构建并回放。
- 依赖固定为言台 `v1.0.0`、言据 1.1.2；ABI v2、平台 1.7、事件 1.3、无障碍 1.0 和绘制
  1.1 契约不变。0.5 只增加向后兼容的文本、数据源和虚拟列表 API。
- Windows、macOS、Linux 的 x86-64 与 ARM64 六目标使用同一份言序源码；文本模型、虚拟列表、
  无窗口集成和真实窗口门禁均在最低与当前工具链上验证。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.5.0", 版 = "^0.5" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并为每个目标重新生成锁文件。

## 升级方式

把依赖修订更新为`v0.5.0`、版本约束更新为`^0.5`，再为每个目标重新生成锁文件。0.1–0.4
的控件、事件、主题和叠层代码无需迁移。需要大集合时，可用新的`数据源`导出和
`容器.虚拟列表（源，配置）`替代手工创建全部行；同步加载器可以逐步迁移为应用自己的异步
调度层，并在取得范围后调用`提供`。

## 制品与校验

Release 附带`yanxu-ui-0.5.0-six-targets.tar.gz`、同名`.sha256`、`yanxu-ui-0.5.0.toml`
完整包清单和 API JSON 快照。六目标候选分别验证包入口、13 个公开示例、文本／数据视图集成、
无窗口集成、真实窗口和所有单元测试；归档保留每个目标锁、自动退出窗口 YXB、数据视图示例
和运行时元数据。Release 只复用与标签同一提交且已通过 CI 的候选。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。0.5.0 不新增直接
原生依赖；数据源和虚拟列表完全由言序实现。

## 已知限制

- 数据源加载器是同步回调；网络、异步调度、取消和重试策略由应用层负责。
- 虚拟列表的无障碍快照提供集合数量、选择和加载诊断，尚未为每个未加载行创建独立原生节点。
- GPU、移动端和 Web 后端不是 0.5.0 的承诺，当前统一使用言台 CPU 二维后端。
