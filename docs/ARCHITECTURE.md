# 言界架构

## 边界

言界不导入 Win32、AppKit、Cocoa、Wayland、X11、Direct2D、CoreGraphics、Metal，也不保存
平台指针。唯一系统边界是言台公开的 ABI v2 值、长期资源、事件批次和二进制绘制帧。

```mermaid
flowchart LR
  A["言序应用"] --> B["言界应用与窗口"]
  B --> C["控件树／布局／事件／文本"]
  C --> D["渲染树与脏区"]
  D --> E["言台二进制绘制协议"]
  B --> F["言台事件批次"]
  E --> G["原生窗口与 CPU 二维后端"]
  F --> C
```

## 一帧的路径

1. 言台把一批平台事件交给`应用.处理批次`。
2. 窗口按窗口编号接收事件；连续指针事件已在言台合并。
3. 指针事件先让窗口级叠层栈处理顶层命中、外部关闭和模态阻断，再按根到目标捕获、目标
   处理、目标到根冒泡。
4. 状态改变调用`使无效`或`请求布局`，脏区管理器合并逻辑像素矩形；托管叠层保留锚框，
   由叠层栈重新计算翻转和夹取。
5. `需要重绘`时才测量／排列必要的树，随后建立保留渲染树；菜单的触发器与所有打开层
   作为一个叠层渲染树提交。
6. 布局或语义变脏时，以稳定控件编号向言台提交完整无障碍树；相同快照由言台去重，菜单
   子项使用窗口内稳定的派生编号并在动作请求时映射回菜单控件。
7. 渲染树按脏区裁剪展开为结构化命令，并由言台一次编码和提交二进制帧；回执替换窗口
   唯一待呈现身份。
8. 匹配的`帧呈现`用进程内单调时间推进活动动画并请求下一帧；终值另提交静态终帧，迟到
   或已替换身份不再驱动状态。

虚拟列表不把数据源的全部项目加入控件树。它根据滚动位置计算可见范围，向`数据源`请求至多
一批项目；数据源先校验批次和返回数量，再原子写入缓存并递增版本。版本事件只使当前视图
重新布局和绘制，旧缓存、焦点和选择身份不会因为邻近范围更新而被重建。

每个绘制原语不会单独跨 ABI；上层也看不到任何原生窗口或渲染句柄。

应用和窗口诊断只汇总编号、数量、状态和有界帧／动画／生命周期计数，不复制控件内容或
语义文字树。平台资源已经释放时，诊断模块根据单向关闭状态直接标记不可用，不再访问原生
句柄；意外查询失败仍转换为结构化稳定错误。

## 输入与焦点

控件树逆序命中最上层可见控件。按下控件可请求指针捕获，保证拖动期间事件仍到原目标。
每窗有独立焦点管理器、快捷键表和 IME 光标区域。键盘事件只表达导航与命令；文字插入只
来自`文本输入`或`IME组合提交`，不会从按键名猜测 Unicode 文字。

言台发回`无障碍焦点请求`时，窗口先提交尚未同步的语义变化，再核对请求携带的树修订，
按稳定编号重新遍历当前控件树。只有仍可见、启用、可聚焦的当前控件会获得焦点；成功后
立即提交新修订，旧树请求不会落到已经关闭或身份变化的控件。

`无障碍动作请求`经过同一修订和编号复核后才交给目标控件。按钮点击和输入框设置值、
UTF-8 字节边界选区、复制、粘贴各自重验状态与参数；处理成功后再提交最终快照。畸形、
越界、旧修订或控件未声明的动作不会降级成普通事件。

## 生命周期

应用拥有窗口、托管计时任务和托管图片；窗口拥有控件树、路由器、焦点、叠层、帧调度和
脏区。控件关闭会递归关闭子树、解除父子关系并清理焦点／捕获。图片资源默认由调用者拥有，
也可用控件配置的`拥有资源`转移所有权；两条路径最终进入同一幂等关闭状态机。

应用创建时先验证言台资源配额和生命周期能力；首个子资源或运行前可以下调配额，之后冻结。
计时器和图片结束后立即解除应用登记，迟到事件和已置空图片句柄不会重新激活资源。窗口按
叠层、帧调度、控件树、无障碍、原生窗口和应用登记最佳努力关闭；应用按计时器、窗口、图片、
监听和原生应用继续收敛。每个步骤独立记录错误，状态只从可用进入关闭中再进入已关闭。
原生资源最终仍按言台的父子资源规则释放。

每个控件创建时取得稳定正整数编号；语义快照直接复用该编号，并明确输出可见、启用、
可聚焦、焦点状态和声明操作。编号只在控件所属窗口的语义树中使用，不作为原生句柄。
