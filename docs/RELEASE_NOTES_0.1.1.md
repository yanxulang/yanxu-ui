# 言界 0.1.1

言界 0.1.1 优化主题样式热路径，并将最低运行时推进到言序 1.1.9。

## 新功能

- 主题对象新增`控件样式`方法，统一取得“控件”基础样式与具体控件种类样式的合并结果。

## 修复内容

- 缓存按样式名称和悬停、按下、焦点、禁用四个状态位解析的主题样式，避免每帧重复解析继承和状态覆盖。
- 缓存控件基础样式与种类样式的深合并结果，减少控件树绘制热路径中的重复分配与遍历。

## 兼容性与平台

- 最低运行时为言序 1.1.9，依赖言台 0.1.0 与言据 1.1.2；
- 继续支持 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标；
- 主题配置格式、已有控件 API、言台协议和言据格式保持兼容。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.1.1", 版 = "^0.1" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并重新构建应用 Bundle。

## 升级方式

先安装言序 1.1.9，再把言界依赖修订更新为`v0.1.1`并重新生成锁文件。主题配置和应用源码
无需迁移。主题对象返回的缓存样式为共享值，调用方应将其视为只读并通过控件实例样式覆盖定制。

## 制品与校验

Release 附带`yanxu-ui-0.1.1-six-targets.tar.gz`、同名`.sha256`、
`yanxu-ui-0.1.1.toml`完整包清单和 API JSON 快照。归档包含六目标锁文件与各目标实际构建的
自动退出窗口示例，并在发布前通过六目标 CI。

## 第三方依赖

本版本不新增或升级第三方依赖，继续使用 MIT OR Apache-2.0 的言台 0.1.0 与 MIT 的言据 1.1.2。

## 已知限制

- 主题初始化后的缓存样式按只读共享值使用，尚未提供运行中局部失效接口；切换主题会创建新的主题对象；
- 跨行选区背景、系统原生无障碍桥、子菜单和 GPU 后端仍沿用 0.1.0 的限制。
