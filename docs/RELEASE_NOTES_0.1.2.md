# 言界 0.1.2

言界 0.1.2 修复当前稳定言序工具链的空值类型推断兼容问题，并把底层平台依赖切换到已
发布的言台 1.0.0 稳定支持线。公开控件 API、主题格式和绘制协议保持不变。

## 新功能

- 发布门禁在六目标最低版本验证之外，增加言序 1.1.20 的源码、单元测试、无窗口集成和
  正式包构建验证。
- 言台依赖升级到 1.0.0，直接使用其 Windows、macOS、Linux x86-64/ARM64 六目标制品、
  稳定错误码和 1.x 兼容政策。

## 修复内容

- 指针与键盘事件不再把初始空值保存在随后写入控件对象的局部变量中，修复言序
  1.1.15–1.1.20 严格空值类型推断下的编译失败。
- 最低与当前工具链的测试卷统一使用 30 秒硬上限，覆盖言台 1.0 冷缓存加载时间，同时
  继续把真正挂起判为失败。

## 兼容性与平台

- 最低运行时仍为言序 1.1.9，并额外验证当前稳定言序 1.1.20；
- 依赖为言台 1.0.0 与言据 1.1.2；ABI v2、平台 1.7、事件 1.3 和绘制 1.1 保持兼容；
- 继续支持 Windows、macOS、Linux 的 x86-64 与 ARM64 六个目标；
- 不改变公开方法、主题配置、事件名称或应用源码，0.1.1 使用者可直接更新锁文件。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.1.2", 版 = "^0.1" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并重新构建应用 Bundle。

## 升级方式

把言界依赖修订更新为`v0.1.2`并重新生成锁文件。言台会从 0.1.0 更新到 1.0.0，必须连同
新锁选中的动态库、SHA-256 和大小一起部署，不能只替换清单或单个二进制。现有主题、控件
配置和应用源码无需迁移。

## 制品与校验

Release 附带`yanxu-ui-0.1.2-six-targets.tar.gz`、同名`.sha256`、
`yanxu-ui-0.1.2.toml`完整包清单和 API JSON 快照。归档包含六目标锁文件与各目标实际构建
的自动退出窗口示例；Release 只复用同一标签提交已经通过的 CI 候选。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。言界主体仍为纯
言序代码，不新增直接原生依赖；言台的依赖、公告和制品来源由其 1.0 Release 记录。

## 已知限制

- 主题初始化后的缓存样式仍按只读共享值使用，运行中局部失效将在后续版本提供；
- 言界已生成语义树，但尚未把控件树提交到言台 1.0 的系统原生无障碍桥；
- 跨行选区背景、子菜单和 GPU 后端仍沿用 0.1 系列限制。
