# 言界 0.6.0

言界 0.6.0 在言台 1.0 的帧提交反馈契约上增加生产级数值动画、单槽渲染背压和内容安全
运行诊断。动画只由真实`帧呈现`事件推进，不使用固定间隔计时器猜测绘制进度；应用可以
持续采样版本化快照，而不读取控件内容或原生句柄。

## 新功能

- `窗口.动画（配置，更新回调）`支持有限起终值、1–3600000 毫秒时长、线性／缓入／缓出／
  缓入缓出和一次性完成回调；任务可查询编号、状态、运行态、取消和内容安全快照。
- 每窗最多 1024 个活动动画，通用队列构造上限为 4096；编号使用不回绕的安全整数空间，
  无效数字、溢出跨度、未知缓动、错误回调和超限时长在注册前失败。
- 帧调度器使用言台`提交缓冲反馈`返回的帧身份和`帧呈现`时间推进任务；新提交会替换唯一
  待呈现身份，迟到或不匹配反馈只累计诊断，不重复推进状态。
- 动画到达终值后额外提交静态终帧；隐藏、最小化或表面暂不可用期间不会积累计时任务，
  恢复重绘时按进程内单调时间追赶。
- 新增`窗口.诊断快照（）`与`应用.诊断快照（）`；原有`应用.调试快照（）`保留为兼容
  别名。快照覆盖布局、交互、无障碍、帧、动画、主题、监听和言台运行状态，并从平台摘要
  移除窗口标题与显示器名称。
- 新增`动画与诊断.yx`公开示例及动画、帧反馈与运行诊断指南。

## 修复内容

- 更新或完成回调抛错时只把所属任务标记为失败，其他动画和事件循环继续运行。
- 回调内取消自己、取消所属队列或关闭窗口不会修改正在遍历的任务列；递归推进和回调内
  新增任务以稳定错误终止当前任务。
- 被替换帧和迟到呈现不再驱动动画；提交失败会累计计数并恢复全量脏区，允许调用方重试。
- 窗口关闭会取消活动任务并清除待呈现身份；关闭后诊断不会访问已释放的平台窗口资源。

## 兼容性与平台

- 最低源码工具链仍为言序 1.1.9；正式制品由言序 1.1.20 在隔离源码副本中构建并回放。
- 依赖继续固定为言台 `v1.0.0` 与言据 1.1.2；ABI v2、平台 1.7、事件 1.3、无障碍 1.0
  和绘制 1.1 契约不变。0.6 只增加向后兼容的窗口动画与诊断方法。
- Windows、macOS、Linux 的 x86-64 与 ARM64 六目标使用同一份言序源码；六目标验证类型、
  单元测试、集成、示例和真实窗口，Linux x86-64 另执行固定 Release 性能预算。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.6.0", 版 = "^0.6" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并为每个目标重新生成锁文件。

## 升级方式

把依赖修订更新为`v0.6.0`、版本约束更新为`^0.6`，再为每个目标重新生成锁文件。0.1–0.5
的控件、事件、主题、文本、数据视图和叠层代码无需迁移。既有计时器驱动动画可以继续运行；
需要和真实窗口呈现同步时，可逐步改用`窗口.动画`并在更新回调中写入控件状态。

## 制品与校验

Release 附带`yanxu-ui-0.6.0-six-targets.tar.gz`、同名`.sha256`、`yanxu-ui-0.6.0.toml`
完整包清单和 API JSON 快照。六目标候选分别验证包入口、14 个公开示例、动画与帧调度单元
测试、无窗口集成、真实窗口动画和资源关闭；归档保留每个目标锁、已回放自动退出窗口 YXB
及运行时元数据。Release 只复用与标签同一提交且已通过 CI 的候选。

Linux x86-64 Release 作业对动画任务生命周期、帧反馈循环、64 个活动任务诊断，以及 32
控件布局与帧编码执行预热、自动校准、9 轮中位数和 MAD 门禁；原始 JSON 报告保留 14 天。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。0.6.0 不新增直接
原生依赖；动画、帧调度和诊断均由言序实现，并复用言台既有反馈事件。

## 已知限制

- 动画当前只插值数值；颜色、路径、关键帧组合、时间线编排和弹簧模型由应用层组合。
- 窗口不可呈现时动画暂停产生反馈，恢复重绘后按单调时间追赶；不承诺后台逐帧回调。
- 性能预算用于固定 CI 镜像的数量级回归，不代表任意机器的显示刷新率或端到端输入延迟。
- GPU、移动端和 Web 后端不是 0.6.0 的承诺，当前统一使用言台 CPU 二维后端。
