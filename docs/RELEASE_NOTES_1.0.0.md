# 言界 1.0.0

言界 1.0.0 是保留模式桌面 GUI 的首个稳定版本。它延续 0.9.0 已冻结的公开 API，不新增
调用面；本次发布最终固定支持政策、十一基线兼容证明、机器发布契约和六目标制品来源。

## 新功能

- 发布 1.x 稳定支持政策，明确补丁、次版本和主版本的兼容边界，以及源码重建、目标锁、
  YXB、诊断快照和无障碍树修订各自的稳定范围。
- 新增`api/release-contract-v1.json`，冻结 API 摘要、上游精确提交、协议版本、两套工具链、
  六个目标、测试规模、Release 附件和支持政策摘要，并由 CI 确定性重建。
- 历史兼容证明扩展到`v0.1.0`–`v0.9.0`十一个已发布基线；1.0 公开 API SHA-256 继续为
  `60596fd0f1c6e4a89cb44acd6109fe40945d386fa617d8f562ee8e8486c1a228`。
- 发布候选报告增加六套根锁／示例锁摘要和 1.0 发布契约摘要，Release 会再次核对报告来源
  提交与归档摘要。

## 修复内容

- 候选演练现在拒绝锁格式、最低生成器、清单摘要、言台／言据精确提交、包数量、言界版本
  或路径来源不一致的目标锁，不再只确认锁文件存在。
- macOS 的候选复现不依赖 GNU `sha256sum`，可回退到系统`shasum -a 256`；即使外部环境
  使用 C locale，中文归档成员也不会被错误计数。
- 归档内示例清单、API 冻结、兼容报告、供应链报告和发布契约均与标签提交逐字节比较，避免
  只替换外部摘要后混入其他候选内容。

## 兼容性与平台

- 最低源码工具链为言序 1.1.9，Release 构建与回放基线为言序 1.1.20；当前稳定工具链继续
  作为独立兼容任务运行。
- 言台固定为`v1.0.0`及提交`9b6bce794a2e23fba04340f762e3d8f49a2724ff`，言据固定为
  1.1.2 提交`765d9dd623db901a3e71aa4759dbcd77563cb3a9`。
- Windows、macOS、Linux GNU 的 x86-64 与 ARM64 六目标保持正式支持；每个目标原生验证
  窗口、事件、CPU 绘制、UIA／NSAccessibility／AT-SPI、锁和候选回放。
- 0.1.0 起的公开签名全部保留；0.9.0 升级到 1.0.0 不需要修改控件调用。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v1.0.0", 版 = "^1.0" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并在每个实际发布目标上独立生成和提交锁文件。

## 升级方式

从 0.9.0 升级只需改用`v1.0.0`与`^1.0`、重新生成各目标锁并重建应用。从更早版本升级时，
按迁移指南处理言台 1.0 锁、显式权限、托管计时器／图片所有权、结构化关闭报告和 0.8 起的
逐项无障碍语义。

升级不能只替换动态库，也不能复制其他目标的锁。上线记录应保存 1.0 标签 CI、Release
附件摘要、目标系统冒烟和必要的回滚锁。

## 制品与校验

Release 附带`yanxu-ui-1.0.0-six-targets.tar.gz`、同名`.sha256`、完整清单快照和 API JSON。
每个目标构建包入口与 15 个公开示例，共 16 个 Release YXB；门禁还覆盖 14 项单元测试、
11 条无窗口集成、5 条真实窗口路径、4096 轮交互、1024 轮资源归零和固定运行器性能预算。

主线、标签和 Release 只传递同一提交的候选。标签 CI 成功包含发布契约重建与独立候选演练；
Release 下载候选报告，核对版本、标签提交和归档摘要后才发布四个公开附件。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。言界自身不携带
DLL、dylib 或 so；平台原生依赖由言台承担。checkout、upload-artifact 与
download-artifact 工作流动作全部固定到 40 位提交。

## 已知限制

- 数据源加载器是同步回调；异步调度、网络权限和取消由应用负责。
- 列表只为当前可见范围生成最多 256 个语义节点；离开当前树的旧节点编号不能复用。
- 菜单快捷键字段只负责显示；全局快捷键由窗口快捷键表注册。
- GPU、移动端、Web、32 位目标和非 GNU Linux 不属于 1.0 支持范围。
