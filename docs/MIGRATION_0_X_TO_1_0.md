# 从言界 0.x 升级到 1.0

言界 1.0 保留 0.1.0 起的公开源码签名。大多数应用只需更新依赖、重新生成每个目标的
锁并重建；需要人工处理的内容主要是 0.7 的资源所有权和 0.8 的集合无障碍语义。

## 通用步骤

1. 提交当前源码和各目标锁，保留可回滚点；
2. 把言界依赖更新到`v1.0.0`并使用`^1.0`约束；
3. 在 Windows、macOS、Linux 的实际目标架构分别重新生成锁，不能复制其他目标的锁；
4. 检查应用只声明实际使用的图形界面、原生扩展、剪贴板和文件对话框权限；
5. 重新构建应用，运行输入法、缩放、主题、屏幕阅读器、文件对话框和关闭路径冒烟；
6. 保存 1.0 标签 CI、Release 附件摘要和应用自己的验收结果。

稳定依赖示例：

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

从 0.9.0 升级时只需切换修订与版本约束、重新生成目标锁并重建；1.0.0 不再改变应用代码。

## 按来源版本处理

| 当前版本 | 升级时重点 |
| --- | --- |
| 0.1.x | 重新生成依赖言台 1.0.0 的锁；补齐显式权限与窗口关闭路径 |
| 0.2.x | 动态主题和原生无障碍已兼容；为自定义控件保留稳定编号、边界和通用状态 |
| 0.3.x | 只读／必填／无效与表单值校验不变；复核错误输入不依赖旧的宽松行为 |
| 0.4.x | 菜单和弹出层统一进入窗口叠层；不要自行维护第二套模态焦点或外部点击状态 |
| 0.5.x | 数据源监听使用返回编号解除；切换数据源时不要比较对象方法身份 |
| 0.6.x | 动画由帧呈现反馈推进；诊断快照是内容安全摘要，不作为业务持久化格式 |
| 0.7.x | 计时器和图片由应用托管；读取结构化关闭报告并明确调用者／控件的图片所有权 |
| 0.8.x | API 已冻结；只需验证锁、制品来源和本应用的目标系统行为 |
| 0.9.x | 候选契约与 1.0 逐字节一致；切换稳定标签、重新生成锁并保存新 Release 来源证明 |

## 0.9 到 1.0

0.9 与 1.0 的公开 API SHA-256 相同。1.0 新增的是稳定支持政策、十一基线兼容证明和机器
发布契约，不新增控件方法或改变事件行为。升级后应确认：

- 锁中的言界版本为 1.0.0，言台仍来自已发布的 1.0.0 精确提交；
- Release 附件包含六目标归档、摘要、清单和 API 快照，且标签 CI 来源与应用记录一致；
- 应用在实际字体、输入法、缩放、屏幕阅读器、主题和关闭路径上完成一次上线冒烟。

稳定版本政策见[支持与稳定性](../SUPPORT.md)，机器来源边界见
[1.0 发布契约](RELEASE_CONTRACT_1_0.md)。

## 资源与关闭

应用创建后、首个子资源或首次运行前可以下调言台配额；冻结后通过关闭不用的资源恢复容量。
`应用.定时器`和`应用.加载图片`返回托管对象，一次计时任务完成后自动解除登记，图片关闭后
句柄置空。窗口和应用`关闭（）`返回结构化报告并继续最佳努力清理后续步骤。

既有代码忽略`关闭（）`返回值仍可运行；生产应用应记录失败步骤，并在应用事件循环结束后
调用一次`应用.关闭（）`。重复关闭是幂等的，不应围绕同一对象编写无限重试。

图片控件默认不拥有调用方传入的资源。只有配置`拥有资源`时，控件关闭才转移并执行资源
关闭；不要同时让调用方和控件假设自己是唯一所有者。

## 列表与无障碍

0.5–0.7 的虚拟列表曾把数量、选择、首可见行、加载状态和错误放进集合语义`值`典，并接收
集合级整数选择。言台无障碍协议只允许标量值，因此 0.8 起：

- 列表集合值是当前选中行的显示文字，未选择为空；
- 诊断分别读取`数据数量／取选择／首可见序／数据状态／数据错误`；
- 当前可见行生成最多 256 个`列表项`节点，选择和滚动到动作属于项目节点；
- 标签页为每个标签生成`标签`节点，选择动作属于标签，不属于集合；
- 辅助请求必须携带当前树修订和当前节点编号，离开树的可见行编号不得缓存复用。

程序代码使用`列表.选择（序）`、`标签页.选择页（序）`和`分割面板.比例值（值）`。不要
直接改写内部状态域或伪造`辅助技术`事件来源。

## 自定义控件

覆写`语义（）`时先调用父实现，保留编号、边界和通用状态。节点`值`只能是空、逻辑、有限
数或有界文字；集合诊断放入控件查询 API，而不是值典。每个声明动作都必须在
`执行无障碍动作`中重新验证可见、启用、关闭状态、参数和当前身份。

公开 API 的历史兼容结果见[0.x–1.0 API 兼容证明](COMPATIBILITY_PROOF.md)，完整上线门禁见
[生产验收](PRODUCTION_VALIDATION.md)。

## 回滚

若目标系统验收失败，恢复原依赖修订和同一目标先前生成的锁，再重新构建；不要只替换言台
动态库或复用新版本 YXB。关闭报告、诊断快照和 CI 链接应随故障记录保留，以区分应用逻辑、
系统配置、字体／输入法资源和平台后端问题。
