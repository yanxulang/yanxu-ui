# 言界 0.1.0

言界是以言序 1.1.8 实现的保留模式桌面 GUI 框架，经言台 0.1.0 在 Windows、macOS 和
Linux 提供统一原生窗口、事件、文字与 CPU 二维绘制。它与既有言窗并行，不要求迁移。

## 新功能

- 控件树、生命周期、测量、行／列／堆叠／网格／滚动／分割布局；
- 命中测试、指针捕获、事件捕获／目标／冒泡、焦点和快捷键；
- 浅色／深色／自定义主题、继承状态样式、言据优先与 JSON 兼容配置；
- 保留渲染树、脏区合并和版本化二进制整帧提交；
- Unicode 单／多行编辑、选区、剪贴板、撤销／重做、导航与中文 IME；
- 文字、按钮、输入框、滚动容器、列表、标签页、分割面板、菜单、弹出层、画布和图片；
- 可观察状态、数据绑定、动画值和无障碍语义树；
- 十一个可构建示例及真实原生窗口自动退出测试。

## 修复内容

这是首个公开版本，没有从旧版继承的修复项。开发期已修正跨模块继承限制、原生回调参数
形状、图片资源构造、事件路由与锁文件更新问题。六目标验收还确认了言序 1.1.7 在 Windows
原生回调路径受默认 1 MiB 主线程栈限制，因此最终版本固定到带 8 MiB 栈修复的言序 1.1.8。

## 兼容性与平台

- 言序 1.1.8；言包 0.5.0；
- 言台 0.1.0；言据 1.1.2；
- `x86_64-pc-windows-msvc`、`aarch64-pc-windows-msvc`；
- `x86_64-apple-darwin`、`aarch64-apple-darwin`；
- `x86_64-unknown-linux-gnu`、`aarch64-unknown-linux-gnu`。

同一言界源码不按操作系统分支。言窗 0.1.x 未修改，现有应用继续使用原路线。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.1.0", 版 = "^0.1" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 装`和`yanbao 查`。构建图形应用 Bundle 使用
`yanbao 构 --release --bundle`。

## 升级方式

这是 0.1 系列起点。后续 0.1.x 更新时修改依赖修订并显式运行`yanbao 更`；不要手工编辑
锁文件或替换言台动态库。

## 制品与校验

Release 附带`yanxu-ui-0.1.0-six-targets.tar.gz`、同名`.sha256`、
`yanxu-ui-0.1.0.toml`完整包清单和 API JSON 快照。归档含六目标锁文件和每个目标实际构建的
自动退出原生窗口示例 YXB；下载后可运行
`sha256sum --check`复核。言台原生动态库及各自 SHA-256 位于言台 0.1.0 Release 和锁文件中。

## 第三方依赖

言界直接依赖 MIT OR Apache-2.0 的言台 0.1.0 与 MIT 的言据 1.1.2。原生窗口、绘制、
文字和图片依赖由言台统一审计，详见`THIRD_PARTY.md`和言台 Release。

## 已知限制

- 跨行选区背景暂为保守矩形；
- 语义树尚未接入系统原生无障碍桥；
- 菜单尚无子菜单，弹出层尚无模态焦点陷阱和屏幕边缘翻转；
- 列表尚未公开异步数据源；
- 当前统一使用 CPU 二维后端，没有 GPU 后端承诺。
