# 生命周期、配额与错误恢复

言界在创建应用时读取言台 1.0 的资源能力，并要求应用级配额、配额下调、拒绝统计、退出
幂等、生命周期统计和关闭资源归零全部可用。协商失败会在创建首个窗口前停止，不根据操作
系统名称猜测能力，也不退回无界资源管理。

## 配额协商

资源配额必须在首次创建窗口、计时器、图片或首次运行应用前配置，之后由言台永久冻结：

```yanxu
定 应用 为 界面.应用（「有界应用」）；

定 能力 为 应用.资源能力（）；
定 生效配额 为 应用.配置资源配额（{
    「资源总数」：64，「窗口数」：4，「计时器数」：16，
    「图片数」：16，「字体数」：8，「图片字节」：67108864，
    「字体字节」：33554432，「帧字节」：33554432，
    「无障碍节点」：16384，「无障碍文字字节」：4194304
}）；
```

省略字段沿用当前值。`资源能力（）`和`资源配额（）`返回复制值，调用者修改结果不会改变
内部状态。未知字段、非整数、超过硬上限或冻结后重新配置由言台稳定拒绝；容量耗尽时保留
原有窗口、任务和资源，不产生半初始化对象。

## 托管计时任务

`应用.定时器（毫秒，重复，回调）`返回托管任务，不再只返回言台计时器包装。任务公开：

- `编号（）`、`状态（）`和`运行中（）`；
- `查询（）`／`快照（）`，只含编号、间隔、重复、状态、触发计数和结构化错误；
- `取消（）`和`关闭（）`，第一次结束返回真，后续调用返回假。

一次任务正常回调后自动进入`完成`并归还平台配额；重复任务保持`运行中`。回调抛错只使
当前任务进入`失败`，错误不会越过事件批次。回调可以取消或关闭自身；已经结束的任务从
应用登记中移除，迟到的计时事件被忽略。应用关闭会先关闭所有仍活动的任务。

## 托管图片资源

`应用.加载图片（内容）`返回托管图片资源。`查询（）`保留既有`宽`、`高`和`字节数`
字段，并增加本地编号、状态和错误；公开`句柄`只用于言台绘制命令。首次`关闭（）`归还
图片数量和解码后 RGBA 字节配额，同时把公开句柄置空，仍引用该资源的图片控件不会再次
提交已关闭句柄。

图片查询或元数据校验失败时，刚创建的言台资源在错误返回前关闭。关闭调用失败会进入
`失败`并保存结构化错误，但仍解除言界登记；最终关闭应用时，言台根资源继续作为兜底。
图片控件默认不取得所有权，配置`拥有资源`时控件关闭会调用同一幂等关闭路径。

## 单向关闭与关闭报告

窗口和应用都按`可用 → 关闭中 → 已关闭`单向迁移。`关闭（）`返回可 JSON 序列化的报告：

```text
名称、状态、成功、步骤总数、成功总数、错误总数、错误
```

每项错误包含`步骤`以及`代码`、`消息`、`来源`。某一步失败不会阻止后续步骤：窗口仍会
依次尝试叠层、帧调度、控件树、无障碍、原生窗口和应用登记清理；应用仍会尝试计时器、
窗口、图片、监听和原生应用清理。重复关闭直接返回原报告，不再次释放资源或调用回调。

`生命周期快照（）`随时返回同结构报告。关闭进行期间的重入调用只观察`关闭中`，不会
启动第二条清理链。应用关闭后，诊断不再查询已释放的言台应用，而是返回`平台可用 = 假`
和空平台错误。

## 诊断与恢复

应用和窗口诊断快照版本为 2，并增加`生命周期`；应用快照还包含`活动图片资源`。快照不
含窗口标题、显示器名称、控件内容、回调或原生句柄，可直接写入结构化日志。

恢复策略应按错误种类决定：

- `PLATFORM_QUOTA_*`：关闭不用的同类资源后重试，或在应用冻结前下调业务并发；
- `UI_TIMER_CALLBACK`：修正回调类型，不创建平台计时器；
- `UI_IMAGE_INFO`：把平台契约不一致视为部署错误，不使用未验证的图片；
- 关闭报告失败：记录全部步骤；应用根关闭已经尝试，不循环重试同一关闭对象。

六目标 CI 在最低和当前言序工具链运行 128 轮窗口、计时器和图片创建／关闭。每轮要求
三类子资源、图片字节、帧字节、无障碍树、节点和文字字节回到零，同时只保留尚未关闭的
应用根资源；最终应用关闭再验证本地报告完成。

完整程序见[`examples/生命周期与恢复.yx`](../examples/生命周期与恢复.yx)。
