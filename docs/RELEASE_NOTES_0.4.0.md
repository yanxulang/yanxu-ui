# 言界 0.4.0

言界 0.4.0 在已发布的言台 1.0 稳定契约上完成窗口级叠层与菜单能力。下拉选择、弹出层和
菜单现在共享同一套有界叠层栈、放置算法、焦点范围、事件命中和无障碍语义；六目标发布门禁
继续使用同一份言序源码和言台 1.0 制品。

## 新功能

- 新增有界弹层放置：检查方向、对齐、尺寸、间隔和视口边距，按四方向候选自动翻转并夹取。
- 新增窗口级叠层栈：顶层绘制和逆序命中、外部点击关闭、Escape 关闭、窗口缩放后的重新布局，
  以及托管叠层锚框保护。
- 新增嵌套模态焦点范围：Tab／Shift+Tab 在当前范围内循环，关闭后恢复触发控件焦点，禁用
  和隐藏控件不会进入焦点序列。
- 下拉选择接入统一叠层，保留选项校验、滚轮、首字匹配、键盘导航和无障碍动作。
- 菜单支持文字或典项目、子菜单、快捷键显示、禁用项目、分隔项目、叶项目动作和保持打开
  选项；单层 1024 项、总数 4096 项、深度 16 层、标题 65536 字符均有界校验。
- 菜单支持指针命中、外部关闭、Escape、Up／Down／Home／End／Left／Right／Enter／空格
  键导航，并为菜单项、分隔符、选中／展开状态和点击／展开／折叠动作生成稳定语义。

## 修复内容

- 叠层外部点击改为调用控件自身命中逻辑，打开的子菜单区域不会被误判为外部区域。
- 菜单和下拉在根布局期间不再被托管布局归零；窗口重排后会重新计算所有打开层。
- 将输入框键盘编辑处理移入表单实现模块，依赖包归档的入口模块从约 33.77 MB 降至约
  30.86 MB，严格保持 YXB 单模块 32 MiB 上限。
- 菜单项目、子菜单、动作和快捷键字段在初始化阶段原子校验，失败不会留下半初始化叠层。

## 兼容性与平台

- 最低源码工具链为言序 1.1.9；正式候选由言序 1.1.20 在隔离源码副本中构建并回放。
- 依赖固定为言台 `v1.0.0`、言据 1.1.2；ABI v2、平台 1.7、事件 1.3、无障碍 1.0 和绘制
  1.1 契约不变。0.4 只增加言界公开能力，不删除 0.1–0.3 的方法、事件或主题字段。
- Windows、macOS、Linux 的 x86-64 与 ARM64 六目标使用同一份言序源码；真实窗口、叠层、菜单
  和原生无障碍动作均在发布门禁中验证。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.4.0", 版 = "^0.4" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并为每个目标重新生成锁文件。

## 升级方式

把依赖修订更新为`v0.4.0`、版本约束更新为`^0.4`，再为每个目标重新生成锁文件。已有
按钮、表单、文本、主题和无障碍代码无需迁移；需要菜单时可继续使用原有`菜单（项目列）`
入口，也可以在项目典中加入`子菜单`、`禁用`、`分隔`、`快捷键`和`动作`字段。快捷键字段只
负责显示，真正的窗口快捷键仍需通过窗口快捷键表注册。

## 制品与校验

Release 附带`yanxu-ui-0.4.0-six-targets.tar.gz`、同名`.sha256`、`yanxu-ui-0.4.0.toml`
完整包清单和 API JSON 快照。六目标候选分别验证包入口、12 个公开示例、9 个单元测试卷、
无窗口集成、真实窗口、叠层窗口和菜单窗口；归档保留目标锁、由言序 1.1.20 构建并实际回放
的自动退出窗口 YXB 及运行时元数据。Release 只复用与标签同一提交且已通过 CI 的候选。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。0.4.0 不新增直接
原生依赖；窗口、绘制、UIA、NSAccessibility 与 AT-SPI 继续由锁定的言台制品提供。

## 已知限制

- 菜单快捷键不自动注册全局快捷键；应用仍需通过窗口快捷键表绑定动作。
- 列表和标签页目前以集合级语义为主，异步数据源和逐项目集合语义留待后续版本。
- 多行跨行选区背景仍使用保守矩形，文本值、IME 和 UTF-8 无障碍选区不受影响。
- GPU、移动端和 Web 后端不是 0.4.0 的承诺，当前统一使用言台 CPU 二维后端。
