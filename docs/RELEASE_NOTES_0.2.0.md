# 言界 0.2.0

言界 0.2.0 在言台 1.0 稳定基线上接通动态主题和系统原生无障碍链路。控件树现在以稳定
编号同步到 UIA、NSAccessibility 或 AT-SPI，并处理经过修订校验的焦点与基础动作请求。

## 新功能

- 主题对象支持原子局部更新、缓存失效和单调修订；应用会通知全部现有窗口重新布局与重绘。
- 语义快照增加稳定正整数编号、实际可聚焦状态和控件操作，同一控件存活期间编号不变。
- 窗口初始化、显示、重绘和关闭路径分别完成无障碍树同步、去重查询与清理。
- 原生焦点请求按当前树修订重新解析控件；成功后立即提交新的焦点快照。
- 按钮接通`点击`动作；输入框接通有界`设置值`、UTF-8 精确`选择`、`复制`和`粘贴`。
- 新增原生无障碍接入指南和六目标真实窗口中的树、焦点、动作与拒绝路径验收。

## 修复内容

- 控件隐藏、禁用或取消可聚焦时清除其直接焦点，避免提交互相矛盾的焦点状态。
- 图层命令使用绘制协议 1.1 的`透明度`字段，修复禁用控件在真实帧编码时被言台拒绝。
- 旧树修订、已关闭节点、禁用控件、未知动作和非 UTF-8 字符边界选区不会落入普通事件。

## 兼容性与平台

- 最低运行时仍为言序 1.1.9，并继续验证当前稳定言序 1.1.20；
- 1.1.9 继续验证源码、测试、锁和最低工具链构建；Release 中的预编译自动关闭 YXB 由
  1.1.20 在六个目标隔离构建并实际回放；
- 依赖保持言台 1.0.0 与言据 1.1.2；ABI v2、平台 1.7、事件 1.3、无障碍 1.0 和绘制 1.1
  均未改变；
- Windows、macOS、Linux 的 x86-64 与 ARM64 六个目标继续使用同一言序源码；
- 0.1 已发布控件、事件、主题和布局 API 保持可用；0.2 只增加方法、字段和协议行为。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.2.0", 版 = "^0.2" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并重新构建应用 Bundle。

## 升级方式

把依赖修订更新为`v0.2.0`、版本约束更新为`^0.2`，再重新生成锁文件。动态局部主题应使用
`应用.更新主题`，不要直接改写主题配置内部值。内置控件会自动同步无障碍树；自定义控件
只声明自己已实现的角色、状态与动作，并保留父实现提供的稳定编号和通用状态。

## 制品与校验

Release 附带`yanxu-ui-0.2.0-six-targets.tar.gz`、同名`.sha256`、
`yanxu-ui-0.2.0.toml`完整包清单和 API JSON 快照。归档包含六目标锁文件及每个目标使用
言序 1.1.20 实际构建、重新打开并运行通过的自动退出窗口 YXB 和运行时元数据；Release
只复用同一标签提交已经通过的 CI 候选。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。言界不新增直接
原生依赖；UIA、NSAccessibility 与 AT-SPI 适配继续由锁定的言台制品提供。

## 已知限制

- 列表、标签页、菜单和弹出层已有基础角色与焦点语义，逐项目集合节点、子菜单和模态焦点
  范围将在后续小版本补齐；
- 多行跨行选区背景仍使用保守矩形，文本值、IME 和 UTF-8 无障碍选区不受影响；
- 列表尚未公开异步数据源；GPU 后端也不是 0.2 的范围，当前统一使用言台 CPU 二维后端。
