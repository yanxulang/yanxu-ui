# 文本与数据视图

言界 0.5.0 在言台 1.0 的文字测量、文字命中和整帧绘制原语之上，提供两类生产级视图：

- 多行输入框按 Unicode 字符边界保存正文，并按行绘制跨行选区；
- 虚拟列表只请求可见范围，使用有界缓存承载增量数据和加载状态。

## 有界数据源

数据源由应用提供总数和一个同步加载器。加载器接收`起点`与`数量`，必须返回恰好相同数量的
列；每次请求最多 256 项，总数最多 100000000 项，默认缓存最多 1024 项。

```yanxu
引「包:言界/数据源」为 数据；

法 加载（起点：数，数量：数）：列 则
    令 结果：列 为 【】；
    逐 偏移 于 范围（0，数量）则
        追加（结果，「第」）；
    终
    归 结果；
终

定 源 为 数据.数据源（100000，加载）；
定 视图 为 窗口.列（{}）.虚拟列表（源，{「首选高」：240}）；
```

`数据源.加载`会把状态置为`加载中`，成功后置为`完成`并递增版本；加载器抛错、返回非列或
返回数量不一致时，状态置为`失败`，错误文本保留在`错误（）`中，已经缓存的数据不会被部分
提交。调用`刷新数据（）`会清空缓存并允许重新请求。

可用的增量操作：

| 操作 | 作用 |
| --- | --- |
| `提供（起点，项目列）` | 原子写入一段已取得的数据并发送`数据更新` |
| `失效（起点，数量）` | 丢弃一段缓存并发送`数据失效` |
| `设置总数（新总数）` | 更新总数并发送`数量变化` |
| `最大缓存（上限）` | 在 1–4096 之间收紧缓存上限 |
| `快照（）` | 返回总数、缓存、水位、状态、错误和版本诊断 |

完整的自动生成签名见[数据源 API 参考](DATA_SOURCE_API.md)。

数据源事件回调只接收不可变语义快照；视图在数据更新、失败、数量变化和缓存清空时自动重新
布局并标记脏区。应用可以监听这些事件来更新加载指示器或重试按钮。

`监听（事件名，回调）`返回单调递增的监听编号；解除订阅时必须把同一事件名和编号传给
`取消监听（事件名，编号）`。编号而不是绑定方法值标识订阅，因此对象方法、闭包和多次绑定
同一回调都可以被精确地独立移除。视图更换数据源时会按编号自动解除旧数据源的四个回调，
不会把旧源的更新带到新视图。

## 虚拟列表

`容器.虚拟列表（源，配置）`只绘制视口前后一行的可见范围。滚轮、布局或数据版本改变时，
视图会请求新的范围；不可用数据显示`等待数据…`，加载期间显示`加载中…`，失败时显示有界
错误文本。选择、焦点和滚动位置不会因为相邻范围更新而失效。

虚拟列表的语义值包含：

- `数量`：数据源总数；
- `选择`：当前行索引，未选择为`-1`；
- `首可见`：当前可见范围起点；
- `加载状态`和`错误`：当前数据源诊断。

辅助技术可以用`设置值`动作选择一个整数行索引；越界、非整数和关闭后的动作都会被拒绝。
行选择仍通过普通`变化`事件回传，事件数据包含索引和值。

数据项为文字时直接显示；其他类型默认显示有界的`[数据]`占位。需要自定义行表示时，在创建
视图后调用`设置行渲染器（回调）`，回调接收数据值和行索引并返回文字。

## Unicode 多行选区

文本模型使用 Unicode 字符索引保存光标和锚点，所有外部 UTF-8 字节范围都必须往返通过精确
字符边界。`字节范围转字符`会拒绝落在多字节字符内部的范围；`设置字节选区`提供同一校验
路径。

`选区行段（行号）`返回该行的局部起止字符位置。多行输入框依据每一行的实际测量宽度生成
独立高亮矩形，因此换行、中文、Emoji 和扩展 Unicode 不会把选区绘制成跨越整块控件的错误
大矩形。滚动时只生成视口附近的行命令，避免长文档在单帧中无界展开。

## 资源与限制

- 数据源不创建线程、原生句柄或隐式网络连接；加载器的调度、权限和取消由应用负责。
- 数据源批次、缓存、文本历史和单个无障碍值均有明确上限；超限输入返回稳定的`UI_DATA_SOURCE_*`
  或`UI_TEXT_RANGE`错误。
- 0.5.0 继续锁定言台 `v1.0.0`，使用 CPU 二维后端；GPU、移动端和 Web 后端不属于本版本
  承诺。
