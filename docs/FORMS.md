# 完整表单指南

言界表单控件共享同一套保留模式控件树、事件路由、主题状态和言台无障碍协议。应用负责
业务校验与提交；控件负责有界状态、输入交互、绘制和语义同步。

## 快速示例

```yanxu
定 姓名 为 布局.输入框（{「占位」：「姓名」，「可访问名称」：「姓名」}）；
姓名.必填（真）；

定 同意 为 布局.复选框（「接受条款」）；
同意.必填（真）；

定 地区 为 布局.下拉选择（【「中国大陆」，「新加坡」，「其他」】）；
地区.可访问名称（「地区」）；

定 音量 为 布局.滑块（{「最小」：0，「最大」：100，「步长」：5，「值」：50}）；
定 完成度 为 布局.进度条（{「最小」：0，「最大」：100，「值」：0}）；
```

完整可运行代码见[完整表单示例](../examples/完整表单.yx)。

## 通用状态

| 状态 | 行为 |
| --- | --- |
| `启用（假）` | 不进入焦点序列并拒绝指针、键盘和辅助技术动作 |
| `只读（真）` | 仍可聚焦和读取；拒绝用户或辅助技术改值，程序方法仍可更新 |
| `必填（真）` | 向协议允许的表单角色公开必填语义；业务是否满足仍由应用判断 |
| `无效（真）` | 向协议允许的角色公开无效语义，并使用危险色反馈 |

输入框切到只读会取消正在进行的 IME 组合。下拉选择切到只读会折叠弹框；滑块切到只读
会释放指针捕获。进度条是不可聚焦、无动作的输出角色，言台协议不允许它声明只读、必填
或无效，因此这些通用状态只影响本地样式，不写入其语义节点。

## 控件行为

### 复选框、单选框和切换

- 复选框支持未选、已检查和混合三态；用户激活后混合态归一为已检查。
- 单选框组名不能为空。同组始终互斥，Tab 只收集选中项或第一个可聚焦项，方向键循环并
  跳过不可修改项。
- 切换支持空格和回车，语义树同时公开检查状态和按下反馈。

### 下拉选择

- 最多 4096 项，每项必须是文字且不超过 65536 UTF-8 字节。
- 支持指针弹框、滚轮、方向键、Home/End、Enter/Space、Escape 和首字匹配。
- `选项（新列）`原子替换数据并尽量保留旧值；返回的`取选项（）`是副本。
- 辅助技术`设置值`接受有效整数索引或精确文字。无效参数不会改变旧值。

下拉选择使用 0.4 的窗口级叠层栈和指针捕获保证外部点击与选项选择一致；窗口边缘自动
翻转、统一叠层排序和 Escape 关闭由叠层能力统一提供。

### 滑块

- `最小`、`最大`、`步长`和`值`必须是有限数；最大值严格大于最小值，跨度和步数也必须
  保持有限。
- 指针、方向键、Home/End、PageUp/PageDown 与辅助技术动作都经过同一夹取和步长吸附。
- `横向`从左到右递增；`纵向`从下到上递增。
- `设置值`、`增加`和`减少`只在可修改状态声明，并在实际值变化时触发`变化`。

### 进度条

- 确定进度公开夹取后的有限数值。
- `不定（真）`公开空值与`忙碌`状态，并绘制有界活动段；切回确定状态继续使用最后一次
  程序设置的值。
- 进度条不进入焦点序列，也不声明设置值或增减动作。

## 变化事件和程序更新

用户或辅助技术成功改变复选、单选、切换、下拉或滑块时触发`变化`。事件数据只包含当前
公开状态，不包含旧树、口令或无关输入。程序调用`已检查／选中／开启／选择／值`等设置
方法不会自动触发变化事件，便于应用恢复状态而不重复执行业务副作用。

应用校验通常在提交时更新`无效`：

```yanxu
法 提交（所事件）则
    定 姓名有效 为 （长度（姓名.取内容（）） 大于 0）；
    定 地区有效 为 （地区.取选择（） 不小于 0）；
    姓名.无效（非 姓名有效）；
    地区.无效（非 地区有效）；
终
```

## 生产建议

- 每个无文字标签的输入控件都设置`可访问名称`。
- 把必填和无效状态绑定到同一业务真源，不从颜色反推校验结果。
- 大型动态选项在进入控件前分页或过滤，不绕过下拉选择容量上限。
- 使用程序方法恢复持久化状态，使用`变化时`处理真实交互。
- 对真实窗口同时验证键盘、指针和辅助技术动作；不要只检查内存字段。
