# 原生无障碍接入

言界把同一棵控件树转换为言台无障碍协议 1.0 的纯言序值，再由言台 1.0 接入 Windows
UIA、macOS NSAccessibility 或 Linux AT-SPI。应用不接触平台对象、句柄或条件分支。

## 生命周期

每个控件创建时取得稳定正整数编号，同一控件存活期间不变。窗口初始化先提交根节点；
控件增删、布局、焦点、内容、启用状态或可访问名称变化时标记语义树为脏。窗口显示或重绘
前提交完整快照，相同快照由言台去重；关闭窗口前清除平台快照。

```yanxu
定 摘要 为 窗口.同步无障碍（）；
定 快照 为 窗口.无障碍快照（）；

# 快照含修订、节点数、文字字节、焦点和规范化的「树」。
```

快照中的`编号`只在当前窗口和树修订内标识控件，不是原生句柄，也不能跨窗口比较。应用
不应保存`查找编号`返回的对象来处理后续平台请求；窗口会在每个请求到达时重新遍历当前树。

每个节点的`值`只使用协议允许的标量：空值、逻辑值、有限数或有界文字。集合数量、加载
状态和错误等诊断不得塞入`值`典；这类信息由控件公开查询 API 或内容安全诊断快照提供。
`状态`仍是协议定义的状态典，`子`仍是节点列。

## 焦点与动作

焦点请求携带节点编号和树修订。窗口先冲刷尚未同步的变化，再要求请求修订与当前修订完全
一致，并确认控件仍可见、启用、可聚焦。成功聚焦后立即提交下一语义修订。

当前内置动作如下：

| 控件 | 动作 | 参数与结果 |
| --- | --- | --- |
| 按钮 | `点击` | 参数为空；触发与指针／键盘相同的`点击`监听 |
| 复选框、切换 | `点击` | 参数为空；切换检查状态并触发`变化` |
| 单选框 | `点击`、`选择` | 参数为空；选择当前项并取消同组旧项 |
| 下拉选择 | `设置值` | 有效整数索引或精确文字；更新值并触发`变化` |
| 下拉选择 | `展开`、`折叠` | 参数为空；同步组合框弹框状态 |
| 滑块 | `设置值` | 有限数；按范围夹取并对齐到步长后触发`变化` |
| 滑块 | `增加`、`减少` | 参数为空；按一个步长调整并保持范围不变量 |
| 进度条 | 无动作 | 确定状态公开有限值；不定状态公开空值与`忙碌` |
| 输入框 | `设置值` | 不超过 65536 UTF-8 字节的文；替换文档并触发`变化` |
| 输入框 | `选择` | `{「起」,「终」}`为当前值的精确 UTF-8 字节边界 |
| 输入框 | `复制`、`粘贴` | 参数为空；复用权限化系统剪贴板 |
| 滚动容器 | `滚动` | 一至两个`横向／纵向`数值增量；偏移夹取到内容范围 |
| 列表 | `滚动` | `纵向`数值增量；集合本身不声明选择动作 |
| 列表项 | `选择`、`滚动到` | 参数为空；选择可见行或把该行移动到视口 |
| 标签 | `点击`、`选择` | 参数为空；切换到对应页面并触发`变化` |
| 分割面板 | `设置值` | 有限比例；夹取到 0.1–0.9 后重新布局 |
| 分割面板 | `增加`、`减少` | 参数为空；每次按 0.05 调整比例 |
| 菜单、菜单项 | `点击`、`展开`、`折叠` | 参数为空；重新校验当前树修订、层级、可见和启用状态 |
| 弹出层 | `展开`、`折叠` | 参数为空；同步叠层、焦点范围和可见状态 |

旧修订、未知编号、已关闭或禁用控件、未声明动作、错误参数和落在多字节字符内部的选区都
会被忽略，不会转成普通事件。设置值参数不会写入诊断或错误说明。

## 自定义控件

自定义控件覆写`语义（）`时先取得父实现的节点，保留其稳定编号、边界和通用状态，再增加
言台协议支持的角色、状态与操作。只有确实实现的操作才能写入`操作`列，并在
`执行无障碍动作（动作，参数）`中重验当前状态和参数；成功返回真，拒绝返回假。

角色、状态、动作集合和硬上限以言台`能力查询（）`及其
[无障碍协议](https://github.com/yanxulang/yanxu-platform/blob/v1.0.0/docs/ACCESSIBILITY_PROTOCOL.md)
为准。言界固定依赖言台 1.x，不根据操作系统名称猜测后端。

## 验收与边界

普通列表和虚拟列表只为当前可见范围生成`列表项`节点，单次最多 256 项；连续快照中仍可见
的同一行复用编号。标签页为每个标签生成`标签`节点，并在标签存活期间保持编号。选择动作
声明在`列表项`或`标签`上，不声明在集合节点上；动作编号在处理时映射回所属控件，再由当前
索引和树修订共同拒绝过期请求。菜单同样把子菜单作为父项目的语义子树提交。

六目标门禁会审计全部公开控件的角色、标量值、状态和动作组合，验证节点编号为正且在树内
唯一，并覆盖滚动、可见列表项、标签、分割面板和弹出层的有效与拒绝路径。真实窗口验收还会
提交完整树、查询规范化快照、处理有效和过期焦点请求，验证运行后端分别报告 Windows UIA、
macOS NSAccessibility 或 Linux AT-SPI，并在关闭后收敛原生桥资源。

言台自身继续负责树深度、节点数、文字容量、角色／状态组合、原生桥生命周期和系统动作参数
的最终校验。模态焦点范围和系统辅助技术桥也由言台负责；没有`原生无障碍桥`能力的非正式
后端不得宣称系统辅助技术已接入。
