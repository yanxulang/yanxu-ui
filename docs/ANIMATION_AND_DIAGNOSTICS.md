# 动画、帧反馈与运行诊断

言界动画使用言台 1.0 已冻结的帧提交回执和`帧呈现`事件，不用固定间隔计时器猜测窗口何时
完成绘制。每个窗口同时只保存一个待呈现帧；应用持续更新时，新帧会有界替换旧帧，不会
形成无界渲染队列。

## 开始和取消动画

`窗口.动画（配置，更新回调）`返回动画任务。配置字段如下：

| 字段 | 类型 | 默认值 | 边界 |
| --- | --- | --- | --- |
| `起值` | 有限数 | `0` | 起终跨度也必须有限 |
| `终值` | 有限数 | `1` | 起终跨度也必须有限 |
| `时长毫秒` | 有限数 | `140` | `1`–`3600000` |
| `缓动` | 文 | `线性` | `线性`、`缓入`、`缓出`、`缓入缓出` |
| `完成` | 法或空 | 空 | 只在正常到达终点后调用一次 |

```yanxu
令 任务 为 空；

法 更新（值） 则
    进度.值（值）；
终

法 完成（） 则
    提示.内容（「完成」）；
终

置 任务 为 窗口.动画（{
    「起值」：0，「终值」：100，「时长毫秒」：600，
    「缓动」：「缓入缓出」，「完成」：完成
}，更新）；

窗口.取消动画（任务）；
```

任务公开`编号（）`、`状态（）`、`运行中（）`、`取消（）`和`快照（）`。状态为`运行中`、
`完成`、`已取消`或`失败`。更新或完成回调抛错时，错误被隔离到该任务，任务转为`失败`，
其他动画和事件循环继续运行；任务快照只保存错误文字，不保存回调对象。回调可以安全取消
自己、取消所属队列或关闭窗口；同一任务的递归推进和在队列推进中新增任务会以稳定错误
结束当前任务，避免修改正在遍历的队列。

每窗最多同时运行 1024 个动画，底层通用队列硬上限为 4096。动画编号是不回绕的安全正
整数；编号空间耗尽、无效数值、未知缓动和超限时长都会在注册前原子失败。

## 帧反馈链

动画的正常顺序为：

1. 注册任务时立即写入起值并请求首帧；
2. `需要重绘`生成完整帧，使用`提交缓冲反馈`取得帧序号；
3. 匹配的`帧呈现`携带进程内单调呈现时间；
4. 言界用该时间推进全部活动任务并请求下一帧；
5. 到达终值后再提交一个静态终帧，终帧呈现后停止调度。

若外部状态在旧帧呈现前触发重绘，言界先按当前单调时间追赶动画，再提交新帧。回执中的
`被替换帧`会替换唯一待呈现身份；旧帧此后不会驱动动画。窗口隐藏、最小化或表面暂不可用
时可以暂停反馈，恢复后的重绘会按当前时间追赶，不积累计时器或帧任务。

提交失败会累计诊断并恢复全量脏区，便于调用方处理错误后重试。动画或完成回调关闭窗口
时，调度器先取消活动任务并清空待呈现身份，关闭路径不会再请求已释放的平台资源。

## 诊断快照

`窗口.诊断快照（）`和`应用.诊断快照（）`返回版本为 2 的纯言序值；既有
`应用.调试快照（）`保留并返回同一应用快照。快照可直接 JSON 序列化，不含回调、原生
句柄、输入框内容、密码值或无障碍文字树。

窗口快照包含：

- 尺寸、关闭状态、布局／语义脏状态和最多 32 个脏区的数量摘要；
- 当前焦点、指针捕获、悬停、焦点范围深度和叠层数量；
- 无障碍修订、节点数和焦点编号，不包含节点内容；
- 单槽待呈现身份、提交／替换／呈现／忽略／失败计数和最后延迟；
- 动画活动数、4096 硬上限内的任务摘要、高水位及完成／取消／失败总数；
- 已移除窗口标题和显示器名称的言台窗口摘要；资源已关闭或查询失败时使用`平台可用`和
  `平台错误`安全降级。
- 单向生命周期状态、关闭步骤数量和结构化关闭错误。

应用快照包含言台完整运行诊断、窗口摘要、主题、活动计时回调、活动图片资源、生命周期和
应用监听摘要。应用已关闭时不会再次查询言台资源，而是直接返回平台不可用且无平台错误。监听
类型名称最多返回 256 项，同时保留真实类型总数、回调总数和`已截断`标记。诊断计数为
进程内累计值，不是持久化协议；监控系统应按时间采样并计算差值，不应修改后回写。

## 性能预算

CI 在固定 Ubuntu x86-64 运行器上使用言序 1.1.20 Release 工具链，分别测量动画任务
生命周期、帧反馈循环、64 个活动任务的诊断快照，以及 32 控件布局与帧编码。每项先预热，
自动把样本校准到至少 25 ms，再记录 9 轮中位数；中位绝对偏差（MAD）不得超过中位数
30%。报告作为`performance-budget`制品保存。

预算用于发现数量级或持续回归，不承诺任意机器的显示延迟，也不替代六目标真实窗口、
资源压力或无障碍测试。修改预算时应保留同机前后报告；放宽阈值必须说明正确性、内存或
兼容权衡。

完整程序见[`examples/动画与诊断.yx`](../examples/动画与诊断.yx)。
