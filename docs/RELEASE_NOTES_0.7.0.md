# 言界 0.7.0

言界 0.7.0 在言台 1.0 已冻结的应用配额和生命周期契约上，增加窗口、计时器和图片的有界
托管、错误恢复与单向关闭报告。资源失败不会留下半初始化登记；关闭中的任一步失败也不会
跳过后续清理。

## 新功能

- 应用创建时验证言台资源配额、配额下调、拒绝统计、退出幂等、生命周期统计和关闭资源
  归零能力；公开`资源能力／资源配额／配置资源配额`，配置结果和硬上限均为复制值。
- `应用.定时器`返回托管任务，支持编号、状态、运行态、查询、内容安全快照、取消和关闭；
  一次任务回调后自动完成，重复任务保持活动，结束后立即解除应用登记。
- `应用.加载图片`返回托管图片资源，保留句柄、宽、高、字节数、查询和关闭兼容接口，并
  增加编号、状态、可用和快照；关闭后句柄置空且归还图片数量和 RGBA 字节配额。
- 窗口和应用`关闭（）`返回版本无关的结构化报告，记录名称、状态、成功、步骤数及逐步骤
  错误；`生命周期快照（）`可在关闭前后读取同一状态，重复关闭不重复执行。
- 应用和窗口诊断升级为版本 2，加入生命周期；应用另报告活动图片资源数，关闭后不再访问
  已释放的言台应用。
- 新增`生命周期与恢复.yx`公开示例，以及配额、托管资源、错误分类和关闭报告指南。

## 修复内容

- 托管回调通过公开登记边界解除自身，避免异步闭包访问类私有成员而把成功关闭误报为失败。
- 计时回调抛错只终止当前任务，迟到事件不再调用已结束任务；取消、关闭和应用收尾均幂等。
- 图片查询或元数据校验失败时关闭刚创建的言台资源；图片关闭失败仍解除本地登记，并阻止
  控件继续提交已关闭句柄。
- 窗口关闭按叠层、帧调度、控件树、无障碍、原生窗口和应用登记继续清理；应用关闭按计时器、
  窗口、图片、监听和原生应用继续收敛，首个错误不会中断后续步骤。
- 生命周期编排、事件批次和资源登记移入独立模块，使新增能力不突破 YXB 单模块 32 MiB
  安全上限。

## 兼容性与平台

- 最低源码工具链仍为言序 1.1.9；正式制品由言序 1.1.20 在隔离源码副本中构建并回放。
- 依赖继续固定为言台 `v1.0.0` 与言据 1.1.2；ABI v2、平台 1.7、事件 1.3、无障碍 1.0
  和绘制 1.1 契约不变，不需要发布新的言台版本。
- 既有计时器和图片调用仍可使用`查询／取消／关闭`或`句柄／查询／关闭`；新增状态与报告是
  向后兼容扩展。窗口和应用关闭现在返回数据典，忽略返回值的既有程序无需修改。
- Windows、macOS、Linux 的 x86-64 与 ARM64 六目标使用同一份言序源码，并执行真实窗口、
  原生无障碍、最低／当前工具链、性能和资源生命周期门禁。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.7.0", 版 = "^0.7" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并为每个目标重新生成锁文件。

## 升级方式

把依赖修订更新为`v0.7.0`、版本约束更新为`^0.7`，再为每个目标重新生成锁文件。0.1–0.6
的控件、事件、主题、文本、数据视图、叠层和动画代码无需迁移。需要主动限制资源的应用应在
创建首个窗口前调用`配置资源配额`；需要记录关闭失败时读取`关闭（）`的返回报告。

计时器和图片现在由言界登记。手动关闭仍被支持且会立即解除登记；应用退出事件循环后建议
调用`应用.关闭（）`完成窗口、任务、图片和监听的统一收尾。

## 制品与校验

Release 附带`yanxu-ui-0.7.0-six-targets.tar.gz`、同名`.sha256`、`yanxu-ui-0.7.0.toml`
完整包清单和 API JSON 快照。六目标候选分别验证包入口、15 个公开示例、14 个单元测试卷、
8 条无窗口集成、4 条真实窗口路径和 Linux x86-64 性能预算；每个目标保存最低工具链锁、
由言序 1.1.20 构建并回放的自动退出 YXB 及运行时元数据。

生命周期压力在最低和当前工具链执行 128 轮窗口、计时器和图片创建／关闭。每轮要求三类
子资源、图片字节、帧字节、无障碍树、节点和文字字节归零，并核对累计创建与关闭只相差仍
存活的应用根。Release 只复用与标签同一提交且已通过 CI 的六目标候选。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。0.7.0 不新增直接
原生依赖；生命周期、登记、诊断和压力验证均由言序实现，并复用言台既有 ABI v2 资源。

## 已知限制

- 配额只能在首个子资源或首次运行前下调；冻结后应通过关闭资源恢复容量，而不是重新配置。
- 关闭报告是进程内快照，不是持久化事务日志；需要审计的应用应在关闭后自行写入外部日志。
- 计时回调在应用事件线程串行执行；耗时或异步工作仍应由应用层调度并显式处理取消。
- GPU、移动端和 Web 后端不是 0.7.0 的承诺，当前统一使用言台 CPU 二维后端。
