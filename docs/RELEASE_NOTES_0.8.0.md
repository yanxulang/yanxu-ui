# 言界 0.8.0

言界 0.8.0 补齐高级控件的生产无障碍语义和六目标验收。滚动容器、普通／虚拟列表、标签页、
分割面板和弹出层现在使用言台 1.0 允许的角色、标量值与动作组合；同一份言序源码在 Windows、
macOS、Linux 的 x86-64 与 ARM64 原生运行器上验证真实窗口、原生无障碍桥和资源收敛。

## 新功能

- 滚动容器把偏移夹取到实际内容范围，支持滚轮和`滚动`无障碍动作；参数只接受一至两个
  `横向／纵向`数值增量，空参数、未知方向和错误类型会被拒绝。
- 普通列表和虚拟列表只为当前可见范围生成最多 256 个`列表项`节点。子节点公开文字、边界、
  选中状态以及`选择／滚动到`动作；集合本身只负责有界滚动，不声明选择动作。
- 列表新增`项目数量／行内容／选择／取选择／滚动到`；虚拟列表沿用同一接口，并在滚动后
  同步首可见行和有界加载范围。
- 标签页为每个标签生成稳定`标签`节点，支持辅助点击／选择，并新增`选择页／取当前页`；
  指针、程序和辅助技术共用同一变化事件路径。
- 分割面板支持方向键、`比例值／取比例`和辅助`设置值／增加／减少`，比例始终夹取到
  0.1–0.9；弹出层公开与当前展开状态对应的展开／折叠动作。
- 新增全控件无障碍审计、六目标生产后端真实窗口验收、4096 轮交互长序列、1024 轮资源
  归零压力，以及可直接复现门禁的生产验收指南。

## 修复内容

- 言台事件循环退出时可能先回收窗口、计时器和图片；言界现在把明确的
  `PLATFORM_RESOURCE_CLOSED`视为已经收敛，不再把后续幂等关闭误报为失败。
- 原生 ABI 的`[PLATFORM_*] 消息`错误格式会归一化为结构化言台错误，关闭报告和诊断不再
  丢失代码与来源。
- 虚拟列表不再让同一滚轮事件同时经过父类和重复处理器；一个增量只应用一次，指针选择仍
  由继承的列表路径正确路由到数据源行。
- 虚拟列表不再把数量、选择、首可见行和加载诊断典写入语义`值`；节点值保持为言台支持的
  空值、逻辑值、有限数或有界文字。
- 语义派生编号、动作映射、滚动测量、标签切换和分割调整等实现细节改为私有，避免进入
  1.0 公开 API；应用不能伪造列表变化事件的内部来源。

## 兼容性与平台

- 最低源码工具链仍为言序 1.1.9；正式制品由言序 1.1.20 在隔离源码副本中构建并回放。
- 依赖继续固定为言台 `v1.0.0` 与言据 1.1.2；ABI v2、平台 1.7、事件 1.3、无障碍 1.0
  和绘制 1.1 契约不变，不要求发布新的底层框架。
- 0.1–0.7 的控件创建方法、事件名称、主题字段和布局单位保持兼容；0.8 新增的方法均为向后
  兼容扩展。直接读取虚拟列表旧语义典的诊断代码需要按下文迁移。
- 六目标全部验证原生架构、窗口事件循环、帧提交与呈现、完整语义树，以及 UIA、
  NSAccessibility 或 AT-SPI 后端标识。

## 安装示例

```toml
[依赖]
言界 = { 包 = "yanxu-ui", git = "https://github.com/yanxulang/yanxu-ui.git", 修订 = "v0.8.0", 版 = "^0.8" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，并为每个目标重新生成锁文件。

## 升级方式

把依赖修订更新为`v0.8.0`、版本约束更新为`^0.8`，再为每个目标重新生成锁文件。普通应用
无需修改控件代码。需要程序选择列表、标签页或分割比例时，优先使用`选择／选择页／比例值`
而不是直接改写公开状态域。

若应用检查虚拟列表无障碍快照，不再从集合`值`读取`数量／选择／首可见／加载状态／错误`典；
改用`数据数量／取选择／首可见序／数据状态／数据错误`。辅助选择应定位当前树中的`列表项`
或`标签`编号并发送其声明的`选择`动作，不再向集合发送整数`设置值`。

## 制品与校验

Release 附带`yanxu-ui-0.8.0-six-targets.tar.gz`、同名`.sha256`、`yanxu-ui-0.8.0.toml`
完整包清单和 API JSON 快照。六目标候选分别验证包入口、15 个公开示例、14 个单元测试卷、
11 条无窗口集成、5 条真实窗口路径和 Linux x86-64 Release 性能预算；包入口与示例共生成
16 个 Release YXB。

交互长序列执行 4096 轮表单、输入、列表、标签、分割、叠层和无障碍操作；生命周期压力
执行 1024 轮窗口、计时器和图片创建／关闭，并要求三类子资源、图片字节、帧字节、无障碍
树、节点和文字字节归零。Release 只复用与标签同一提交且已经通过 CI 的六目标候选。

## 第三方依赖

言台 1.0.0 采用 MIT OR Apache-2.0 双许可，言据 1.1.2 采用 MIT 许可。0.8.0 不新增直接
原生依赖；高级控件语义、动作映射、审计和压力验证均由言序实现，并复用言台既有 ABI v2
和系统无障碍桥。

## 已知限制

- 数据源加载器仍是同步回调；异步调度、网络权限和取消由应用层负责。
- 可见列表项离开当前语义树后，旧编号不能用于后续动作；辅助技术必须使用当前树修订。
- 菜单快捷键字段只负责显示，不自动注册全局快捷键；全局快捷键由窗口快捷键表负责。
- GPU、移动端和 Web 后端不是 0.8.0 的承诺，当前统一使用言台 CPU 二维后端。
