# 生产验收

言界 0.9.0 把“生产可用”定义为：同一份言序源码在六个正式桌面目标通过兼容、真实窗口、
无障碍、性能、压力、资源收敛、制品重放和发布完整性门禁，并能证明候选公开 API、历史
兼容和供应链证据未漂移。单个平台上的示例窗口成功不等于通过本页的生产基线。

## 支持矩阵

| 目标 | 运行器 | 原生无障碍后端 |
| --- | --- | --- |
| `x86_64-pc-windows-msvc` | Windows 2025 x86-64 | UIA |
| `aarch64-pc-windows-msvc` | Windows 11 ARM64 | UIA |
| `x86_64-apple-darwin` | macOS 15 Intel | NSAccessibility |
| `aarch64-apple-darwin` | macOS 15 ARM64 | NSAccessibility |
| `x86_64-unknown-linux-gnu` | Ubuntu 24.04 x86-64 | AT-SPI |
| `aarch64-unknown-linux-gnu` | Ubuntu 24.04 ARM64 | AT-SPI |

每个矩阵任务先核对 Rust 宿主架构与目标完全一致，不以交叉编译代替原生运行。Linux 使用
Xvfb 提供隔离显示环境；窗口、事件循环、CPU 二维绘制和无障碍桥仍走正式言台后端。

## 门禁层次

### 源码与兼容

- 言序 1.1.9 对`src／tests／integration／examples`全部言序文件执行格式幂等和静态检查；
- 14 项单元测试与 11 条无窗口集成在六个目标执行；
- 言序 1.1.20 重新解析依赖、静态检查、执行同一组单元与无窗口集成，并构建正式包入口；
- API JSON、生成参考文档、依赖边界、本地文档链接、许可、安全和第三方声明必须一致。

### 真实窗口与生产后端

六目标分别运行五条真实窗口路径：基础窗口与无障碍请求、叠层、菜单、动画反馈和生产后端。
生产后端路径必须确认 ABI v2、平台 1.7、事件 1.3、无障碍 1.0、绘制 1.1，以及原生窗口、
复杂文字、高 DPI、帧反馈和原生无障碍桥能力。事件循环至少消费一个原生事件，窗口至少提交
并呈现一帧，完整语义树必须进入言台。

全控件无障碍审计遍历每个节点，要求编号为正且树内唯一，角色属于言台 1.0 词汇，`值`只为
空、逻辑、有限数或文字，并逐项核对角色允许的状态和动作。滚动容器、普通／虚拟列表项、
标签、分割面板和弹出层另有有效参数、越界夹取、错误参数和过期身份回归。

### 压力与资源收敛

- `交互长序列.yx`执行 4096 轮表单、输入、列表、标签、分割、叠层和无障碍交互，检查稳定
  编号、有界状态以及最终关闭；
- `生命周期压力.yx`执行 1024 轮应用、窗口、计时器和图片生命周期，要求应用、窗口、三类
  子资源、图片字节、帧字节、无障碍树、节点和文字字节全部归零；
- 两条长测试显式使用`YANXU_MAX_STEPS=100000000`，避免把合理压力工作量误判为失控程序；
- Linux x86-64 Release 性能任务执行四个固定场景，要求报告版本、Release 模式、总结果和每个
  场景都通过预算。

### 制品与发布

每个目标使用言序 1.1.9 构建包入口和 15 个公开示例，共 16 个 Release YXB；随后在干净的
归档目录中用言序 1.1.20 重新解析、构建并实际运行自动关闭示例。六目标候选只在全部兼容、
性能和政策任务成功后汇总。

版本发布必须依次满足：合并提交的主线 CI 成功、版本标签指向该提交、标签 CI 成功、发布
工作流核对标签与 CI 的提交完全一致。Release 只复用该次标签 CI 的候选，公开六目标归档、
SHA-256 校验、清单快照和 API JSON，不在发布阶段重新生成未经矩阵验收的二进制。

### 发布候选演练

六目标汇总完成后，独立候选任务重新下载四个待发布附件并运行
`scripts/verify-release-candidate.sh`。演练拒绝以下任一情况：

- 附件不是恰好一个六目标归档、一个归档 SHA-256、一个完整清单和一个 API 快照；
- 外部 SHA-256、清单、API 快照、1.0 候选冻结或机器发布契约与当前提交不一致；
- 归档包含越界路径、符号链接、硬链接或其他特殊文件；
- 任一正式目标缺少根锁、示例锁、自动关闭 YXB、运行时元数据或内部 SHA-256；
- 锁中的目标、运行时版本、运行时目标、YXB 摘要或归档内 API／兼容／供应链证据不一致。

通过后上传 14 天保留的`release-candidate-report`，记录来源提交、归档／清单／API 摘要、
六套根锁与示例锁摘要，以及六个目标 YXB 的摘要与字节数。该报告是候选完整性证明，不替代
标签 CI；正式 Release 仍只接受与标签提交相同的成功标签 CI run。

## 本地复现核心门禁

在仓库根目录设置`YANXU_BIN`为待验证工具链后，可复现不依赖当前系统之外目标的核心部分：

```sh
core=${YANXU_BIN:-yanxu}
"$core" 包 更新 .
"$core" 包 更新 examples
"$core" 试 tests --超时 30000
"$core" 字节 integration/无障碍全控件.yx
YANXU_MAX_STEPS=100000000 "$core" 字节 integration/交互长序列.yx
YANXU_MAX_STEPS=100000000 "$core" 字节 integration/生命周期压力.yx
YANXU_BIN="$core" sh scripts/verify-api.sh
sh scripts/check-boundary.sh
node scripts/check-doc-links.mjs
```

下载某次成功 CI 的`yanxu-ui-six-targets`附件后，可以复现发布候选演练：

```sh
CANDIDATE_DIR=/path/to/yanxu-ui-six-targets \
REPORT=/tmp/yanxu-ui-release-candidate.txt \
SOURCE_SHA=local \
bash scripts/verify-release-candidate.sh
```

真实窗口应再运行`真实窗口／叠层窗口／菜单窗口／动画窗口／生产后端窗口`五条集成。Linux
无桌面会话时使用`xvfb-run -a`。本地结果只能证明当前宿主；六目标结论以同一提交的正式矩阵
为准。

## 应用上线清单

- 固定言界版本标签，并提交目标平台生成的锁文件；
- 只申请实际使用的图形界面、原生扩展、剪贴板和文件对话框权限；
- 为图片、计时器、窗口和应用安排明确的关闭所有权，记录结构化关闭报告；
- 为自定义控件提供有界语义名称、协议允许的标量值和真正实现的动作；
- 对业务所需字体、输入法、缩放、屏幕阅读器、系统主题和文件对话框做目标系统验收；
- 保存上线提交、标签 CI 链接、Release 附件校验和应用自己的冒烟结果。

## 不在 0.9 承诺内

GPU、移动端和 Web 后端不属于 0.9 支持范围。数据源加载器仍是同步回调，异步调度、网络
权限和取消由应用负责。菜单快捷键字段只负责显示；全局快捷键由窗口快捷键表注册。
